# Step 1 resource changelog

## 2.0.0-pilot.2 — 2026-07-15

- 整合领域专家（经济学博士）整理的《关键词0715》；原 DOCX 不进入仓库，以文件 SHA-256 和结构化纳入清单追踪来源；
- 在既有“法规描述词 + 高水平论文技术词”结构内合并专家词，不新增路由类别；
- 新增现行《网络安全法（2025 修正）》来源，并加入网络安全、个人/身份信息、数据风险、网络攻击、系统漏洞、访问授权等种子词；
- 将“数据、信息、数据库、图像采集”等宽泛词保留为上下文或待验证候选，不允许单独产生 S；
- 真实专利 smoke test 显示“身份识别 + 普通业务信息”会产生取餐场景误报，因此“身份识别”不进入生产词；
- 将虚假信息/名誉等内容治理词和“病毒”单词设为诊断项，避免扩大到内容治理或生物领域；
- 新增 `expert_keywords_0715.json`，逐项校验实际词表、上下文和诊断规则与专家纳入决策一致；
- 本版本仍是等待人工开发集与留出集验证的 pilot 专家种子词表。

## 2.0.0-pilot.1 — 2026-07-15

- 将旧 S/W/R/E 词表改为 S/E 路由；
- 描述性词和技术词分别记录，但任一有效命中均路由为 S；
- 密码学、加密、密钥、隐私计算等基础技术不再要求额外出现“保护数据”；
- 通用安全词和邻近技术必须通过局部上下文；
- IPC 只输出审计命中，不改变 S/E；
- 增加方法论文、数据安全专利研究和逐项来源追踪；
- 根据首批真实专利 smoke test，取消“监测/审计/认证”等通用词仅凭数据对象共现形成 S，将“安全/保护”等通用词改为必须关联具体数据对象，并为歧义缩写 DLP 增加安全目标或威胁上下文门槛；
- 本版本仍是等待人工开发集与留出集验证的 pilot 种子词表。

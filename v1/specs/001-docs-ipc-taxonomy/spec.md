# Spec 001: 数据安全专利 DOCS/IPC 四级体系

- 状态：Accepted
- 版本：1.1.0
- 日期：2026-07-11
- 范围：仅建设分类依据，不实现分类算法

## 1. 目标

为中国上市公司专利建立可解释、可追溯、可版本化的领域证据体系。体系分别对：

1. `DOCS`：`摘要文本`与`主权项内容`拼接后的专利技术文本；
2. `IPC`：`IPC分类号`与`IPC主分类号`中的有效 IPC 号；

给出强相关（S）、弱相关（W）、泛相关（R）和不相关（E）四级证据。后续分类算法必须基于本体系融合 DOCS 与 IPC，不得把单一关键词或单一 IPC 号直接视为最终专利标签。

## 2. 非目标

本阶段不包含：

- 文本向量模型、文本相似度模型或 LLM 分类器；
- 训练集、验证集和测试集构建；
- 相似度阈值、四分类分数或最终融合公式；
- 对完整专利 CSV 执行分类；
- 论文实证变量构建。

## 3. 四级语义

| 等级 | 名称 | 证据语义 | 允许的解释 |
| --- | --- | --- | --- |
| S | 强相关 | 文本或 IPC 直接指向数据、个人信息或隐私的保护机制 | 高精度领域证据，但仍需上下文核验 |
| W | 弱相关 | 指向隐私、去标识化、合规、备份、完整性、审计、可信执行等支撑机制 | 进入模型候选，但不可单独形成正类结论 |
| R | 泛相关 | 指向通用密码、认证、网络、应用、云或邻近安全场景 | 用于扩大候选召回，不可单独形成正类结论 |
| E | 未路由 | 未识别出 S/W/R 词项或 IPC 规则 | 不等于最终不相关；按分层概率抽样进入模型与人工复核 |

等级描述的是“候选路由强度”，不是最终分类结果。`S > W > R > E` 仅用于候选优先级。
最终标签只有“明确数据安全相关”“可能数据安全相关但不确定”“其他”三类。

## 4. DOCS 数据契约

### 4.1 输入字段

```text
DOCS = 摘要文本 + "\n[主权项]\n" + 主权项内容
```

- 任一字段缺失时使用另一字段，不补写不存在的文本。
- `专利名称`只用于展示、抽样和人工复核，不计入 DOCS 匹配。
- 不把 IPC 文本、申请人、行业或公司名称混入 DOCS。
- 清除首尾空白与重复空白，但保留句子边界和否定词。

### 4.2 词项要求

`config/taxonomy/docs_taxonomy.json` 中每个实际用于匹配的词项必须：

- 属于一个且仅一个 S/W/R/E 词组；
- 至少关联一个 `source_id`；
- 说明是可独立使用，还是必须与对象、动作、目标或威胁上下文共现；
- 对高歧义词说明上下文要求；不得通过硬排除词降低路由等级。

### 4.3 上下文单位

上下文检测的默认最小单位为同一句；无可靠标点时，使用命中词左右各 48 个中文字符的窗口。窗口参数属于后续算法配置，本规格只固定“必须检测上下文”的要求。

实现要求：上下文检索不能只返回一个布尔值。每个关键词命中应关联保存上下文词表 ID、上下文类型、
实际命中词、标准化文本位置、与关键词的字符距离、证据片段、来源 ID，以及本次使用的检索范围
（`sentence` 或 `window`）。`cooccurrence` 词以关联结果执行 `required_any` 门控；独立词项也保留
上下文关联，便于后续复核其保护对象、生命周期动作、安全目标和威胁。

## 5. IPC 数据契约

- 同时读取 `IPC分类号`和`IPC主分类号`，去重后判断。
- 标准化空格、大小写和全角字符，保留完整小组号。
- 首版同时支持 WIPO IPC 2021.01 与 2026.01，兼容 2021 年申请专利及现行分类。
- 采用“最具体规则优先”：子组规则覆盖父组规则。
- 一件专利有多个 IPC 号时，保留全部命中及其来源，不只保留最高级。
- 未命中任何规则的 IPC 默认为 E。
- CPC、FI 等非 IPC 号不得假定与 IPC 等价；需要单独版本映射。

## 6. 来源要求

来源优先级如下：

1. 中国法律、行政法规；
2. 现行国家标准、行业标准、TC260 指南；
3. WIPO IPC 主文件；
4. 原始论文或正式期刊综述；
5. 本项目专利样本中的可复现误判观察。

`config/taxonomy/sources.json` 是来源注册表。每个来源必须包含标题、发布者、年份、URL 或本地附件说明、具体条款/页码及其对体系的贡献。

## 7. 路由与边界规则

1. `keyword_level` 和 `ipc_level` 分别取实际命中的最高等级，`route_level` 再取二者最高等级。
2. S/W/R 全量进入 LLM 三分类，不由静态词表直接生成最终标签。
3. E 不等于不相关。E 按年度、行业、IPC 部和文本完整性分层抽样进入 LLM/人工复核，并记录样本权重。
4. `生产安全`、`行车安全`、`食品安全`等仅作事后诊断信号，不得作为前置硬排除或覆盖 S/W/R。
5. `数据`、`数据库`、`数据处理`、`大数据`等通用数据词不创建候选，只作为上下文对象。
6. `加密`、`认证`、`密钥`、`哈希`、`数字签名`、网络安全等通用安全词属于 R。
7. 联邦学习、同态加密、安全多方计算、隐私计算等专门技术属于 S；隐私保护、匿名化、去标识化、备份、完整性、安全存储、审计和可信执行环境属于 W。
8. IPC-S 是路由强先验而非最终正类；模型仍须判断专利核心技术对象。

## 8. 可审计输出要求

后续算法对每件专利至少应输出：

- `keyword_level`、`ipc_level`、`route_level`和最终三分类 `cat`；
- 命中的词项 ID、IPC 规则 ID、上下文片段；
- 使用的 taxonomy 版本和来源 ID；
- 发生升级、降级或冲突处理时的规则 ID；
- 模型、阈值和处理时间（在后续规格中定义）。

## 9. 验收标准

- [ ] `sources.json` 中 source ID 唯一。
- [ ] DOCS 的每个词组和上下文词表均有有效来源。
- [ ] 同一关键词不会同时出现在多个等级。
- [ ] IPC 的每条规则均标明 WIPO 版本与来源。
- [ ] 关键强相关词和 `G06F 21/60`、`G06F 21/62`、`G06F 21/64` 有固定回归测试。
- [ ] `H04L 9/00`、身份认证、区块链等不会被静态定义为 S；联邦学习固定为 S。
- [ ] 默认 IPC 等级为 E。
- [ ] 物理安全诊断词不会降低任何 S/W/R 路由。
- [x] 上下文关联检索优先同句、无句界时退回 48 字窗口，并保留可审计关联明细。
- [ ] 本阶段不出现模型推理或最终分类实现。

## 10. 决策来源

本版已与附件《数据安全专利识别方案（结论版）》对齐。DOCS 静态路由只使用摘要和主权项；专利名称、DOCS 和 IPC 均可作为后续 LLM 判断证据。
